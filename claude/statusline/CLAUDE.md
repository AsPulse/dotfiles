# claude-statusline

## テスト時の注意

`cargo run` でテストデータを流すと `~/.local/share/claude-statusline/rate_limits.db` に書き込まれる。テスト用の `resets_at` が本来ありえない値だと、ウィンドウ切り替えの合成レコードが不整合な状態で挿入され、以降の通常実行で重複レコードが無限に増殖する。

テストする場合は環境変数 `HOME` を一時ディレクトリに向けて、本番の DB に書き込まないようにすること。

万が一本番 DB を汚染してしまった場合は、不正なレコードを手動で特定・削除する必要がある。ただし不正レコードが原因で通常実行時にも重複データが大量生成されるため、正常なデータの一部が失われる可能性がある。
